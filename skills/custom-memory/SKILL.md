# Custom Memory Manager

Persistent memory system with project-scoped and global memory stores.

## Project Memory (default)

Memory is stored in the current project directory under `<project-name>-memory/`.

| Command | What it loads |
|---------|--------------|
| `/custom-memory load short` | `<project>-memory/<project>-short-memory.md` — rolling window |
| `/custom-memory load long` | `<project>-memory/<project>-long-memory.md` — full history |
| `/custom-memory load important` | `<project>-memory/<project>-important-memory.md` — flagged moments |
| `/custom-memory load all` | All three project memory files |
| `/custom-memory status` | Daemon status (project + global) |
| `/custom-memory refresh` | Manually trigger daemon |

## Global Memory (all projects)

Prefix with `all` to access cross-project memory stored at `~/.claude/memory/`.

| Command | What it loads |
|---------|--------------|
| `/custom-memory all load short` | `~/.claude/memory/short-memory.md` |
| `/custom-memory all load long` | `~/.claude/memory/long-memory.md` |
| `/custom-memory all load important` | `~/.claude/memory/important-memory.md` |

## Behavior

### When user invokes `/custom-memory load <type>` (no `all`):

1. Determine the current project name from the working directory (last segment of cwd)
2. Look for `<project-name>-memory/` in the project root
3. Read the corresponding `<project-name>-<type>-memory.md` file
4. Use the loaded context to inform responses about past work on THIS project

### When user invokes `/custom-memory all load <type>`:

1. Read from `~/.claude/memory/<type>-memory.md`
2. This contains sessions from ALL projects

### When user says "load memory" without specifics:

Load **project short memory** first (most recent, most relevant). Offer to load long or important if needed.

### If the memory file doesn't exist:

Tell the user: "No memory recorded yet for this project. Run `python3 ~/.claude/memory/memory-daemon.py` or wait for the next cron cycle."

### `/custom-memory status`:

Run: `python3 ~/.claude/memory/memory-daemon.py --status`

### `/custom-memory refresh`:

Run: `python3 ~/.claude/memory/memory-daemon.py --config ~/.claude/memory/memory.conf`

## Notes

- Memory files are auto-generated by the daemon — do not edit them manually
- The daemon runs on a cron schedule (configurable via `memory.conf`)
- To flag something as important, use `/important` instead
- Project memory dirs (`<name>-memory/`) should be added to `.gitignore`

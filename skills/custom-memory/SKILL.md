# Custom Memory Manager

You have access to a persistent memory system with three memory stores.

## Memory Stores

| Store | File | Purpose |
|-------|------|---------|
| **Long** | `~/.claude/memory/long-memory.md` | Full history across all sessions, append-only |
| **Short** | `~/.claude/memory/short-memory.md` | Rolling window (default 3 hours) of recent work |
| **Important** | `~/.claude/memory/important.md` | User-flagged critical moments |

## Commands

### `/custom-memory load long`
Read `~/.claude/memory/long-memory.md` and use it as context for the current conversation.

### `/custom-memory load short`
Read `~/.claude/memory/short-memory.md` for recent session context.

### `/custom-memory load important`
Read `~/.claude/memory/important.md` for user-flagged important moments.

### `/custom-memory load all`
Read all three memory files. Use when the user needs full project context.

### `/custom-memory status`
Run: `python3 ~/.claude/memory/memory-daemon.py --status`
Report the output to the user.

### `/custom-memory refresh`
Run: `python3 ~/.claude/memory/memory-daemon.py --config ~/.claude/memory/memory.conf`
This manually triggers the daemon to update memory files from latest transcripts.

## Behavior

When the user invokes `/custom-memory load <type>`:

1. Read the corresponding memory file using the Read tool
2. Parse the content — each `## date — project` section is one session entry
3. Use the loaded context to inform your responses about past work, decisions, and patterns
4. If the file doesn't exist or is empty, tell the user: "No memory recorded yet. Run `python3 ~/.claude/memory/memory-daemon.py` or wait for the next cron cycle."

When the user says "load memory" or "recall" without specifying a type, load **short** memory first (most recent and relevant), then offer to load long or important if needed.

## Important Notes

- Memory files are auto-generated by the daemon — do not edit them manually
- The daemon runs on a cron schedule (configurable via `memory.conf`)
- To flag something as important, use `/important` instead
- Memory is per-machine, stored at `~/.claude/memory/`
